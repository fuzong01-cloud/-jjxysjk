# 头雁学员信息管理系统 AGENTS.md

## 项目目标

基于《头雁学员名单2024》Excel 建立最终版学员信息管理系统，实现：

1. Excel 一键导入数据库。
2. 学员基本信息管理。
3. 受教育情况管理。
4. 荣誉情况管理。
5. 新型经营主体情况管理。
6. 近三年营收情况管理。
7. 主营产业情况管理。
8. 个人培育信息管理。
9. 按身份证号、姓名、所在区县、经营主体名称查询。
10. 新增个人荣誉。
11. 修改个人基础信息。
12. 修改经营主体信息。
13. 查询结果导出 Excel。
14. 导入错误报告。
15. 操作日志。
16. 登录、角色、权限控制。
17. 数据库备份与恢复。
18. API 文档与部署文档。

## 推荐技术栈

- Python 3.11+
- FastAPI
- SQLAlchemy 2.x
- Alembic
- Pydantic v2
- SQLite 用于本地开发
- PostgreSQL 用于正式环境
- pandas / openpyxl 用于 Excel 导入与导出
- pytest 用于测试
- Jinja2 / Bootstrap 或简单前端用于管理后台
- Docker / Docker Compose 用于部署

## 核心业务规则

1. 身份证号是学员唯一业务标识。
2. 姓名只用于检索，不允许作为唯一主键。
3. 年龄不应作为主要持久化字段，应由出生日期动态计算。
4. 荣誉信息必须拆成多条明细记录。
5. 近三年营收数据必须按年份拆成明细记录。
6. 主营产业必须支持多条记录。
7. 经营主体与学员之间应建立清晰关系。
8. 所有导入记录必须保留原始 Excel 行号。
9. 导入时不得静默丢弃数据，所有异常都应进入导入错误报告。
10. 修改数据前必须定位到唯一学员。
11. 涉及修改、删除、导入、导出、登录等操作必须记录审计日志。
12. 接口返回值必须结构化，错误信息要明确可读。
13. 数据库字段命名使用 snake_case。
14. Python 代码必须包含类型标注。
15. 关键功能必须有 pytest 测试。
16. 文档必须能指导非原作者完成本地启动、导入数据、查询和部署。

## 数据模块建议

系统应至少包含以下业务模块：

- student_basic_info：学员基本信息
- education_experience：受教育情况
- honor_record：荣誉情况
- business_entity：新型经营主体情况
- annual_revenue_record：近三年营收情况
- main_industry：主营产业情况
- cultivation_info：个人培育信息
- import_batch：导入批次
- import_error：导入错误
- user_account：系统用户
- role / permission：角色权限
- audit_log：操作日志
- backup_record：备份记录

## 开发方式

Codex 在开始实现前，应先阅读本文件和 `.agents/skills` 下的相关 `SKILL.md`。

遇到复杂任务时，先输出方案和文件清单，再修改代码。

不要一次性生成无法验证的大量代码。每个阶段实现后应说明：

1. 修改了哪些文件。
2. 实现了哪些功能。
3. 如何运行。
4. 如何测试。
5. 后续还缺什么。
